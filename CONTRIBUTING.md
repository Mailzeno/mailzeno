# Contributing to MailZeno

Thank you for your interest in contributing to MailZeno 🚀

MailZeno follows an open-core model. Contributions are welcome for
the open-source components and documentation.

---

## Where You Can Contribute

We welcome improvements in:

- `packages/core`
- `packages/client`
- `apps/docs`
- Documentation clarity
- Bug fixes
- Performance improvements
- Developer experience enhancements

---

## Areas Maintained by Core Team

The following areas are maintained at the discretion of the MailZeno team:

- SaaS-layer business logic (`apps/web`)
- Abuse prevention systems
- Billing and subscription logic
- Infrastructure policies
- Security enforcement mechanisms

Pull requests in these areas may not be accepted.

---

## Development Setup

1. Fork the repository
2. Install dependencies

```bash
npm install

npm run dev
```
---

## Guidelines

- Keep changes minimal and focused

- Follow existing code style

- Ensure TypeScript types are preserved

- Avoid breaking API behavior without discussion

- Open an issue before large changes
